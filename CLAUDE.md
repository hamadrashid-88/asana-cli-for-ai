# CLAUDE.md

Zero-dependency Asana CLI for AI agents. Single Node.js file, no `node_modules`.

## Entry point

- `asana-cli.js` — the entire CLI. Run with `node asana-cli.js <command> [options]`. All output is JSON for machine parsing.
- Requires **Node.js 18+** (uses native `fetch`).

## Configuration

- **`.env`** (gitignored) — `ASANA_ACCESS_TOKEN` + `ASANA_WORKSPACE_GID`. Each user creates their own.
- **`config.json`** (gitignored) — optional friendly aliases for teams/projects/sections/priorities. Auto-generated by `node asana-cli.js setup`. Template at `config.example.json`.
- The CLI works without `config.json` — just use raw GIDs everywhere.

## Skill

The CLI ships with `.claude/skills/asana-management/SKILL.md`. It teaches Claude (and other agents) the command surface, discovery workflow, and auto-run vs. approval policy. When working in this repo or anywhere that imports this skill, that's the operational guide.

## Distribution note

This repo is itself a product — downstream users copy the SKILL.md into their own projects. As of the Claude Code migration, the canonical location is `.claude/skills/asana-management/SKILL.md`. The README still references the legacy `.agents/skills/` path for Antigravity/Gemini users; if you keep both ecosystems, mirror or symlink as needed.

## Important

- Read commands (`my-tasks`, `task get`, `project list`, etc.) are safe to auto-run.
- Write commands (`task create`, `task update`, `task delete`, `task comment`, etc.) require explicit approval before execution.
